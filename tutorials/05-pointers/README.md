# 第五課：指標（Pointers）

> **一句話總結**：指標就是「房子的地址」—— 有了地址，你不用搬整棟房子，也能找到房子、修改裡面的東西。

## 適合哪個階段的工程師？

| 階段 | 說明 |
|------|------|
| 🟢 初學者 | **重要基礎**：理解 & 和 * 的用法 |
| 🟡 中級工程師 | 了解何時用指標（避免複製、修改原值） |

## 你會學到什麼？

- `&`（取址）和 `*`（解引用）運算子
- 傳值（pass by value）vs 傳指標（pass by pointer）的差異
- 結構體指標和 Go 的語法糖（`ptr.Field`）
- `new()` 函式的用途
- `nil` 指標和為什麼一定要檢查
- 為什麼 Repository 回傳 `*User` 而不是 `User`
- **【重點】** 切片（Slice）和 Map 的內部指標機制
- 什麼時候「不要」用指標

## 指標語法速查表

| 語法 | 意義 | 範例 |
|------|------|------|
| `*int` | 「指向 int 的指標」型別 | `var p *int` |
| `&x` | 取得 x 的記憶體位址 | `p := &x` |
| `*p` | 透過指標取得值（解引用） | `fmt.Println(*p)` |
| `nil` | 指標的零值（不指向任何東西） | `if p == nil { ... }` |
| `new(T)` | 分配 T 的記憶體，回傳 `*T` | `p := new(int)` |
| `ptr.Field` | 語法糖，等同 `(*ptr).Field` | `userPtr.Name` |

### 記憶口訣

```
& → 取「地址」（& 長得像 A，Address 的 A）
* → 取「值」  （穿透指標，取出裡面的東西）
```

## 記憶體模型圖

### 基本指標

```
  變數 x              指標 p
┌─────────┐       ┌─────────────┐
│  42      │ ◄──── │  0xc000...  │
│  (int)   │       │  (*int)     │
└─────────┘       └─────────────┘
  房子                房子的地址

  p := &x    // p 儲存了 x 的位址
  *p         // 透過位址找到 x，取出 42
  *p = 100   // 透過位址找到 x，把值改成 100
```

### 傳值 vs 傳指標

```
  傳值（Pass by Value）：
  ┌──────┐    複製    ┌──────┐
  │  10  │ ────────► │  10  │  ← 函式收到的是副本
  │ main │           │ func │    修改副本不影響原始值
  └──────┘           └──────┘

  傳指標（Pass by Pointer）：
  ┌──────┐    傳地址   ┌──────────┐
  │  10  │ ◄───────── │ 0xc000.. │  ← 函式收到的是地址
  │ main │            │   func   │    透過地址修改原始值
  └──────┘            └──────────┘
```

## 【重點】切片（Slice）的內部結構

這是初學者最容易搞混的部分！切片不是陣列，切片其實是一個包含三個欄位的結構體：

### Slice Header 結構圖

```
  切片變數 nums             底層陣列
┌──────────────────┐     ┌───┬───┬───┬───┬───┐
│ ptr  ─────────────────►│ 1 │ 2 │ 3 │ 4 │ 5 │
│ len = 5          │     └───┴───┴───┴───┴───┘
│ cap = 5          │
└──────────────────┘
   Slice Header
   （這整個會被複製）     （這個不會被複製，是共享的）
```

等同於：

```go
type slice struct {
    ptr *array  // 指向底層陣列的指標
    len int     // 目前有多少元素
    cap int     // 底層陣列最多能放多少元素
}
```

### 為什麼切片不用 `&` 就能在函式內修改？

```
  main()                    函式()
┌────────────┐           ┌────────────┐
│ ptr ───────────┐       │ ptr ───────────┐
│ len = 3    │   │       │ len = 3    │   │
│ cap = 3    │   │       │ cap = 3    │   │
└────────────┘   │       └────────────┘   │
  原始 header     │         複製的 header   │
                  ▼                        ▼
              ┌───┬───┬───┐
              │ 1 │ 2 │ 3 │  ← 同一個底層陣列！
              └───┴───┴───┘

所以：函式內的 nums[0] = 99  → 外面也變成 99 ✅
```

### 為什麼 `append()` 後一定要用 `=` 接收？

```
  append 前（容量不夠）：
  ┌────────────┐       ┌───┬───┬───┐
  │ ptr ──────────────►│ 1 │ 2 │ 3 │  ← 舊陣列（滿了）
  │ len = 3    │       └───┴───┴───┘
  │ cap = 3    │
  └────────────┘

  append 後（Go 建立新的更大陣列）：
  ┌────────────┐       ┌───┬───┬───┬─────┬───┬───┐
  │ ptr ──────────────►│ 1 │ 2 │ 3 │ 999 │   │   │  ← 新陣列！
  │ len = 4    │       └───┴───┴───┴─────┴───┴───┘
  │ cap = 6    │
  └────────────┘

  如果不用 = 接收，外面的 header 還指向舊陣列：
  main 的 header → 舊陣列 [1, 2, 3]      ← 看不到 999
  函式的 header → 新陣列 [1, 2, 3, 999]  ← 只有函式看得到
```

**結論**：`append` 一定要這樣寫：

```go
nums = append(nums, newItem)  // ← 用 = 把新的 header 存回來！
```

## 【重點】Map 的內部結構

Map 比切片更簡單：map 變數本身就是一個指向雜湊表（hash table）的指標。

```
  map 變數 scores           雜湊表
┌──────────────┐         ┌─────────────────┐
│  *hmap ──────────────►│ "Alice" → 90    │
│              │         │ "Bob"   → 85    │
└──────────────┘         └─────────────────┘
  這是一個指標              實際儲存資料的地方
```

所以：
- 傳入函式時，函式收到的是**指標的副本**，但指向**同一個雜湊表**
- 函式內的修改，外面都看得到
- **不需要** `&` 也不需要回傳
- 和切片不同，map 沒有 `append` 的問題

## 決策指南：什麼時候用指標？什麼時候用值？

| 情況 | 建議 | 原因 |
|------|------|------|
| 需要修改原始資料 | 用指標 `*T` | 傳值只會修改副本 |
| 大型結構體（很多欄位） | 用指標 `*T` | 避免複製大量資料 |
| Repository 回傳值 | 用指標 `*T` | 可以用 `nil` 表示「不存在」 |
| GORM 的 `db.Create()` | 用指標 `&user` | GORM 需要回寫 ID |
| 小型結構體（2-3 個欄位） | 用值 `T` | 複製成本低，更安全 |
| 只讀取不修改 | 用值 `T` | 保證不會被意外修改 |
| 基本型別（int, string） | 用值 | 本身就很小 |
| 切片和 Map | 直接傳 | 內部已有指標 |

## 部落格專案中的應用

### 1. Repository 回傳 `*User`

```go
// 找到 → 回傳 &user（指標）
// 沒找到 → 回傳 nil
func (r *userRepository) FindByID(id uint) (*User, error) {
    var user User
    err := r.db.First(&user, id).Error
    if err != nil {
        return nil, err  // nil 代表找不到
    }
    return &user, nil    // & 取址，回傳指標
}
```

### 2. GORM 需要指標來回寫 ID

```go
user := &domain.User{Username: "alice"}  // 建立指標
db.Create(user)                           // GORM 透過指標把自動產生的 ID 寫回 user.ID
fmt.Println(user.ID)                      // 可以讀到 GORM 設定的 ID
```

### 3. 方法接收者用指標

```go
func (r *userRepository) Create(user *User) error {
//      ^                        ^
//      |                        |
//      指標接收者：               指標參數：
//      避免複製 repository       讓 GORM 能修改 user
    return r.db.Create(user).Error
}
```

## 常見問題（FAQ）

### Q: `*` 什麼時候是「型別」，什麼時候是「解引用」？

```go
var p *int   // 這裡的 * 是型別的一部分：「指向 int 的指標」
*p = 42      // 這裡的 * 是運算子：「透過指標存取值」
```

看位置就知道：出現在**型別位置**就是指標型別，出現在**表達式**裡就是解引用。

### Q: 什麼時候用 `new()` vs `&T{}`？

```go
p1 := new(User)                    // 所有欄位都是零值
p2 := &User{Name: "Alice"}         // 可以同時初始化欄位
```

**建議**：大多數情況用 `&T{}` 更好，因為可以同時設定初始值。`new()` 比較少用。

### Q: 切片要不要用 `*[]int`？

**幾乎不需要**。切片內部已經有指標了，直接傳 `[]int` 就好。
除非你需要在函式內 `append` 並且不想用回傳值，才需要 `*[]int`。

### Q: 為什麼有時候看到 `db.Find(&users)` 要加 `&`？

GORM 的 `Find` 需要指標才能把查詢結果寫入你的變數：

```go
var users []User
db.Find(&users)  // & 取得 users 的指標，GORM 透過指標寫入資料
```
