## 用生活來理解

### 陣列 = 固定大小的書架

你買了一個 3 格的書架，只能放 3 本書，多一本都放不下：

```
[書A] [書B] [書C]     ← 滿了！不能再加
```

### 切片 = 可以伸縮的書架

這是一個神奇的書架，放不下的時候會自動變大：

```
[書A] [書B] [書C]     ← 目前 3 本
[書A] [書B] [書C] [書D] [  ] [  ]  ← 加了一本，書架自動變成 6 格
```

### Map = 電話簿

電話簿讓你用「名字」快速找到「電話號碼」：

```
電話簿（Map）
┌──────────┬──────────┐
│ 名字(key) │ 電話(value) │
├──────────┼──────────┤
│ "Alice"  │ 25       │
│ "Bob"    │ 30       │
│ "Carol"  │ 28       │
└──────────┴──────────┘
```

## Array vs Slice 比較表

| 特性 | 陣列（Array） | 切片（Slice） |
|------|-------------|-------------|
| 長度 | 固定，宣告時決定 | 可變，可以動態增減 |
| 宣告 | `[3]int{1,2,3}` | `[]int{1,2,3}` |
| 型別 | `[3]int`（長度是型別的一部分） | `[]int` |
| 傳入函式 | 複製整個陣列 | 只複製 header（指標+長度+容量） |
| 能用 append? | ❌ 不行 | ✅ 可以 |
| 實際使用頻率 | 極少（約 1%） | 幾乎都用（約 99%） |

```go
// 陣列：[3]int 和 [5]int 是不同的型別！
var a [3]int = [3]int{1, 2, 3}

// 切片：[]int 不管多長都是同一個型別
var s []int = []int{1, 2, 3}
```

## 切片的內部結構

切片本質上是一個包含三個欄位的「描述符」（descriptor）：

```
切片變數                          記憶體中的底層陣列
┌──────────────────────┐         ┌───┬───┬───┬───┬───┐
│ pointer → ──────────────────→ │ 1 │ 2 │ 3 │ 4 │ 5 │
│ length  = 5          │         └───┴───┴───┴───┴───┘
│ capacity = 5         │
└──────────────────────┘

s := []int{1, 2, 3, 4, 5}
len(s) = 5    ← 目前有 5 個元素
cap(s) = 5    ← 底層陣列能容納 5 個元素
```

當你從切片中切出一段時，新的切片和原始切片**共享同一個底層陣列**：

```
original := []int{1, 2, 3, 4, 5}
sub := original[1:3]

original                              底層陣列
┌──────────────────────┐         ┌───┬───┬───┬───┬───┐
│ pointer → ──────────────────→ │ 1 │ 2 │ 3 │ 4 │ 5 │
│ length  = 5          │         └───┴─↑─┴───┴───┴───┘
│ capacity = 5         │               │
└──────────────────────┘               │
                                       │
sub                                    │
┌──────────────────────┐               │
│ pointer → ───────────────────────────┘
│ length  = 2          │  ← 只有 [2, 3] 兩個元素
│ capacity = 4         │  ← 從索引 1 到底層陣列末尾 = 4
└──────────────────────┘

⚠️ 修改 sub[0] 會同時影響 original[1]！
```

## make() 詳解

`make()` 用來建立切片（和 Map），可以指定初始長度和容量：

```go
// make([]型別, 長度, 容量)
s1 := make([]int, 5)       // 長度 5，容量 5（預設和長度一樣）
                            // → [0 0 0 0 0]（5 個零值）

s2 := make([]int, 0, 10)   // 長度 0，容量 10
                            // → []（空的，但已預留 10 個位置）

s3 := make([]int, 3, 10)   // 長度 3，容量 10
                            // → [0 0 0]（3 個零值，還能再放 7 個）
```

### 常見錯誤

```go
// ❌ 錯誤：make([]int, 5) 會建立 5 個零值元素
wrong := make([]int, 5)       // [0 0 0 0 0]
wrong = append(wrong, 1)      // [0 0 0 0 0 1] ← 不是 [1]！

// ✅ 正確：make([]int, 0, 5) 建立空切片，預留 5 個位置
right := make([]int, 0, 5)    // []
right = append(right, 1)      // [1] ← 這才對！
```

## append() 的擴容行為

當切片容量不夠時，`append()` 會自動擴容：

```
新增元素  →  長度  →  容量（大約以 2 倍成長）
第 1 個      1       1
第 2 個      2       2
第 3 個      3       4      ← 擴容！
第 5 個      5       8      ← 擴容！
第 9 個      9       16     ← 擴容！
```

每次擴容都要：
1. 分配一塊更大的記憶體
2. 複製所有舊資料
3. 回傳指向新陣列的切片

**效能建議**：如果你事先知道大約需要多少元素，用 `make([]T, 0, cap)` 預先分配容量，避免多次擴容。

## Map 常用操作速查表

| 操作 | 語法 | 說明 |
|------|------|------|
| 建立（字面值） | `m := map[K]V{"a": 1}` | 直接初始化 |
| 建立（make） | `m := make(map[K]V)` | 空的 Map |
| 新增/修改 | `m["key"] = value` | key 不存在就新增，存在就修改 |
| 存取 | `val := m["key"]` | 不存在回傳零值 |
| 存在性檢查 | `val, ok := m["key"]` | ok 為 true 表示存在 |
| 刪除 | `delete(m, "key")` | 刪除不存在的 key 不會報錯 |
| 長度 | `len(m)` | 回傳 key-value 對的數量 |
| 遍歷 | `for k, v := range m` | ⚠️ 順序隨機 |

### nil Map vs 空 Map

```go
var m1 map[string]int           // nil map（未初始化）
m2 := map[string]int{}          // 空 map（已初始化，0 個元素）
m3 := make(map[string]int)      // 空 map（已初始化，0 個元素）

// 讀取 nil map → 不會報錯，回傳零值
_ = m1["key"]                   // 0

// 寫入 nil map → panic！程式崩潰
// m1["key"] = 1                // ❌ panic: assignment to entry in nil map

// 寫入空 map → 正常
m2["key"] = 1                   // ✅ 沒問題
```

> **規則**：使用 Map 之前一定要初始化！用 `make()` 或字面值 `map[K]V{}`。

## Slice 常用操作速查表

| 操作 | 語法 | 說明 |
|------|------|------|
| 建立 | `s := []int{1, 2, 3}` | 字面值初始化 |
| make | `s := make([]int, 0, 10)` | 預分配容量 |
| 存取 | `s[0]` | 索引從 0 開始 |
| 長度 | `len(s)` | 目前的元素數量 |
| 容量 | `cap(s)` | 底層陣列的大小 |
| 新增 | `s = append(s, 4)` | 一定要接回來！ |
| 切片 | `s[1:3]` | 包含 1，不包含 3 |
| 複製 | `copy(dst, src)` | 建立獨立副本 |
| 遍歷 | `for i, v := range s` | 取得索引和值 |

## 在部落格專案中的應用

### Slice 的應用

```go
// domain/article.go
type Article struct {
    ID       uint
    Title    string
    Content  string
    Tags     []string      // 一篇文章有多個標籤 → 用 Slice
    Comments []Comment     // 一篇文章有多個留言 → 用 Slice
}

// 查詢結果回傳文章列表
func FindAll(query ArticleQuery) ([]Article, int64, error)
//                                 ^^^^^^^^
//                                 回傳 Article 的 Slice
```

### Map 的應用

```go
// 種子資料（用 Slice 儲存初始文章）
var seedArticles = []Article{
    {Title: "Go 入門", Content: "...", Tags: []string{"go", "beginner"}},
    {Title: "Go Web",  Content: "...", Tags: []string{"go", "web"}},
}

// 測試中用 Map 模擬資料庫
type mockUserRepository struct {
    users map[string]*domain.User    // key=用戶名, value=用戶資料
}

// ArticleQuery 用來傳遞查詢參數
type ArticleQuery struct {
    Page    int       // 第幾頁
    Limit   int       // 每頁幾筆
    Tag     string    // 篩選標籤
    Keyword string    // 搜尋關鍵字
}
```

## 常見問題（FAQ）

### Q1: 什麼時候用 Slice，什麼時候用 Map？

| 場景 | 用什麼 | 原因 |
|------|--------|------|
| 有序的一組資料 | Slice | 保持順序，支援索引存取 |
| 用 key 快速查找 | Map | O(1) 的查找速度 |
| 去重 | Map | 利用 key 的唯一性 |
| 統計次數 | Map | `counts[key]++` 超方便 |
| 需要排序 | Slice | Map 無法排序 |

### Q2: append 為什麼要用 s = append(s, x) 接回來？

因為 append 可能會：
- 在原始底層陣列上加元素（容量夠的時候）→ 回傳相同的指標
- 建立新的底層陣列（容量不夠的時候）→ 回傳不同的指標

如果不接回來，擴容後你的變數還指向舊的陣列，新加的元素就丟了。

### Q3: Map 可以用什麼型別當 key？

Map 的 key 必須是**可比較的型別**（comparable）：
- ✅ 可以：`string`、`int`、`float64`、`bool`、`struct`（如果所有欄位都可比較）
- ❌ 不行：`slice`、`map`、`func`（這些不能用 `==` 比較）

### Q4: 怎麼從 Slice 中刪除元素？

Go 沒有內建的刪除函式，常見做法：

```go
// 刪除索引 i 的元素（不保持順序，效能較好）
s[i] = s[len(s)-1]    // 把最後一個元素放到被刪除的位置
s = s[:len(s)-1]       // 切掉最後一個元素

// 刪除索引 i 的元素（保持順序）
s = append(s[:i], s[i+1:]...)
```

### Q5: 空切片和 nil 切片有什麼差別？

```go
var s1 []int            // nil 切片（s1 == nil 為 true）
s2 := []int{}           // 空切片（s2 == nil 為 false）
s3 := make([]int, 0)    // 空切片（s3 == nil 為 false）
```

功能上幾乎沒差——`len()`、`cap()`、`append()`、`for range` 對三者的行為都一樣。但序列化成 JSON 時有差異：nil 切片會變成 `null`，空切片會變成 `[]`。

## 練習

1. **去重函式**：寫一個 `removeDuplicates(nums []int) []int` 函式，用 Map 去除重複的數字
2. **平均分數**：建立一個 `map[string]int` 儲存學生分數，計算全班平均分數
3. **反轉切片**：實作 `reverse(s []int) []int`，回傳反轉後的新切片（不修改原始切片）
4. **分組統計**：給定一組文章，用 Map 按作者分組，統計每個作者的文章標題列表
