## 用生活來理解

### 變數 = 盒子

想像你有很多盒子，每個盒子上都有**名字**：

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│  name    │  │   age    │  │  height  │
│ "Alice"  │  │   25     │  │  165.5   │
│ (文字)   │  │ (整數)   │  │ (小數)   │
└──────────┘  └──────────┘  └──────────┘
```

- **變數名稱**（name、age）= 盒子上的標籤
- **值**（"Alice"、25）= 盒子裡裝的東西
- **型別**（string、int）= 這個盒子「只能裝什麼」

### 型別 = 盒子的規格

Go 是「靜態型別」語言，意思是：**每個盒子在建立時就要決定它能裝什麼**，而且之後不能改。

- `string` 盒子只能裝文字
- `int` 盒子只能裝整數
- 你不能把文字塞進 `int` 盒子裡（Go 會在編譯時就告訴你錯了）

這跟 JavaScript、Python 不一樣——它們的盒子什麼都能裝（動態型別）。

## Go 的 4 種基本型別

| 型別 | 中文 | 能裝什麼 | 範例 | 零值 |
|------|------|----------|------|------|
| `string` | 字串 | 文字 | `"hello"`、`"你好"` | `""`（空字串） |
| `int` | 整數 | 沒有小數點的數字 | `42`、`-10`、`0` | `0` |
| `float64` | 浮點數 | 有小數點的數字 | `3.14`、`165.5` | `0` |
| `bool` | 布林 | 只有「是」或「否」 | `true`、`false` | `false` |

> **為什麼叫 float64？**
> "float" 是浮點數（floating point），"64" 代表用 64 個位元來儲存，精度比較高。另外還有 `float32`（32 位元，精度較低），但一般都用 `float64`。

## 三種建立變數的方法

```go
// 方法 1：var + 型別（最完整，像填表格一樣每格都填）
var name string = "Alice"

// 方法 2：var + 省略型別（Go 會看值自動判斷型別）
var name = "Alice"

// 方法 3：:=（最常用！最簡潔！只能在函式裡面用）
name := "Alice"
```

### 什麼時候用哪一種？

| 情境 | 建議用法 | 原因 |
|------|---------|------|
| 在函式裡面宣告 | `:=` | 最簡潔，Go 工程師最常用 |
| 在函式外面宣告（全域） | `var` | `:=` 不能在函式外面用 |
| 只宣告不給值（要用零值） | `var x int` | `:=` 一定要給值 |
| 一次宣告很多個 | `var ( ... )` | 用小括號包起來比較整齊 |

## 零值（Zero Value）

```go
var x int    // x 自動是 0
var s string // s 自動是 ""
var b bool   // b 自動是 false
```

> **為什麼 Go 要有零值？**
> 很多語言（如 C）如果你宣告了變數但沒給值，裡面會是「亂七八糟的垃圾資料」，這很危險。Go 的設計者認為：**每個變數從出生的那一刻起，就應該有一個安全、可預測的值**。所以 Go 會自動幫你填入零值。

## Go 的整數家族

Go 不是只有一種 `int`，它有一整個家族：

### 有號整數（可以是負數）

| 型別 | 能存的範圍 | 佔多少記憶體 |
|------|-----------|------------|
| `int8` | -128 ~ 127 | 1 byte |
| `int16` | -32,768 ~ 32,767 | 2 bytes |
| `int32` | 約 -21 億 ~ 21 億 | 4 bytes |
| `int64` | 超級大的數字 | 8 bytes |
| `int` | 跟電腦位元數一樣（通常 = int64） | 4 或 8 bytes |

### 無號整數（只能是 0 或正數）

| 型別 | 能存的範圍 | 佔多少記憶體 |
|------|-----------|------------|
| `uint8`（= `byte`） | 0 ~ 255 | 1 byte |
| `uint16` | 0 ~ 65,535 | 2 bytes |
| `uint32` | 0 ~ 約 42 億 | 4 bytes |
| `uint64` | 超級大的正數 | 8 bytes |
| `uint` | 跟電腦位元數一樣（通常 = uint64） | 4 或 8 bytes |

> **在部落格專案中**，資料庫的 ID 用 `uint`（無號整數），因為 ID 不可能是負數。
> 你會在 `internal/domain/user.go` 看到 `ID uint`。

## fmt 套件——你的第一個好朋友

`fmt`（讀作 "fumpt"）是 Go 內建的「格式化輸入輸出」套件。

### 三個最重要的函式

| 函式 | 功能 | 會不會印到螢幕 | 會不會換行 |
|------|------|--------------|-----------|
| `fmt.Println(...)` | 印出來 | 會 | 會自動換行 |
| `fmt.Printf(...)` | 格式化印出 | 會 | 不會（要自己加 `\n`） |
| `fmt.Sprintf(...)` | 格式化成字串 | 不會（回傳字串） | — |

### Printf 的佔位符（像填空題）

```go
name := "Alice"
age := 25

// %s 會被 name 的值取代，%d 會被 age 的值取代
fmt.Printf("我叫 %s，今年 %d 歲\n", name, age)
// 印出：我叫 Alice，今年 25 歲
```

| 佔位符 | 用途 | 範例 |
|--------|------|------|
| `%s` | 字串（string） | `fmt.Printf("%s", "hello")` → `hello` |
| `%d` | 整數（decimal） | `fmt.Printf("%d", 42)` → `42` |
| `%f` | 浮點數（float） | `fmt.Printf("%f", 3.14)` → `3.140000` |
| `%.2f` | 浮點數（小數 2 位） | `fmt.Printf("%.2f", 3.14)` → `3.14` |
| `%v` | 任何型別都行（value） | `fmt.Printf("%v", true)` → `true` |
| `%T` | 印出型別名稱（Type） | `fmt.Printf("%T", 42)` → `int` |
| `%t` | 布林值 | `fmt.Printf("%t", true)` → `true` |
| `\n` | 換行 | — |

## 常見問題

### Q: `:=` 和 `=` 有什麼不同？
- `:=` 是「**建立**新變數 + 賦值」（第一次用）
- `=` 是「**更新**已存在的變數」（之後要改值時用）

```go
name := "Alice"  // 建立新變數 name
name = "Bob"     // 更新 name 的值（注意：沒有冒號）
```

### Q: 為什麼 Go 不自動轉型別？
Go 的設計哲學是「**明確大於隱含**」。自動轉換可能導致難以發現的 bug（例如 int 轉 float 時精度遺失）。強制你手動轉換，讓每次轉換都是「有意識的決定」。

### Q: `var` 和 `const` 的差別？
- `var` 宣告的變數**可以改值**
- `const` 宣告的常數**不能改值**（改了會編譯錯誤）
